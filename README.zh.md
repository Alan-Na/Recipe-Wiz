# Recipe Wiz

[English](./README.md) | **中文**

## 项目简介

Recipe Wiz 是一款全栈膳食规划与食谱管理 Web 应用。用户可以按食材搜索食谱、分析营养成分、规划每周菜单，并借助 AI 一键生成个性化膳食计划 —— 全程支持中英文双语切换。

## 核心功能

- **智能食谱搜索**
  - 根据手边食材快速查找食谱。
  - 按饮食标签、健康标签、菜系类型筛选结果。
  - 搜索结果跨页面导航持久保留，不会意外丢失。

- **营养分析**
  - 每道食谱提供详细营养素分解（卡路里、蛋白质、脂肪、碳水、维生素、矿物质）。
  - 色码标签便于快速浏览关键营养素。

- **个性化膳食计划**
  - 以周为单位的日历视图，支持早餐 / 午餐 / 晚餐 / 点心四个餐次。
  - 每天已有餐食时，自动在日期卡片上显示当日热量合计（🔥 千卡）。
  - 可更新每餐状态：已计划 → 进行中 → 已完成。
  - 自由添加、移除、重新安排餐食。

- **已保存食谱库**
  - 一键保存搜索到的食谱。
  - 在膳食计划面板中直接删除不需要的食谱。
  - 已保存食谱可直接加入周历安排。

- **AI 膳食规划** *(由 DeepSeek 驱动)*
  - 输入性别、年龄、身高、体重及健康目标。
  - AI 仅使用您已保存的食谱，自动生成从今天到本周日的完整膳食计划。
  - 支持预览后再确认 —— 不满意可一键重新生成，满意后一键写入日历。
  - 至少需要 3 个已保存食谱才可启用。

- **灵活调整份量**
  - 任意缩放食谱份数，食材用量自动重新计算。

- **中英双语界面**
  - 完整支持中文 / 英文，运行时一键切换，语言偏好跨会话持久保存。

## 快速开始

### 环境要求

- Java 17+
- Node.js 18+
- Maven 3.8+

### 1 — 克隆仓库

```bash
git clone https://github.com/Alan-Na/Recipe-Wiz.git
cd Recipe-Wiz
```

### 2 — 启动后端

```bash
cd backend
mvn package -DskipTests
java -jar target/recipewiz-backend-0.1.0-SNAPSHOT.jar --server.port=9271
```

> 默认监听端口 **9271**，可通过 `--server.port=<端口>` 覆盖。

### 3 — 启动前端

```bash
cd frontend
cp .env.example .env.local   # 或手动创建（见下文）
npm install
npm run dev -- --port 9272
```

`.env.local` 内容：

```
VITE_API_BASE_URL=http://localhost:9271
```

访问 **http://localhost:9272** 即可使用。

### 环境变量（可选覆盖）

| 变量名 | 默认值 | 说明 |
|---|---|---|
| `RECIPE_APP_ID` | 内置 | Edamam 食谱搜索 App ID |
| `RECIPE_APP_KEY` | 内置 | Edamam 食谱搜索 App Key |
| `NA_APP_ID` | 内置 | Edamam 营养分析 App ID |
| `NA_APP_KEY` | 内置 | Edamam 营养分析 App Key |
| `DEEPSEEK_API_KEY` | 内置 | DeepSeek API Key（AI 膳食规划功能） |

## 使用指南

### 食谱搜索

1. **添加食材** — 在输入框中输入食材名称，点击**添加**或按 Enter 键。
2. **筛选条件** — 点击**筛选**打开筛选抽屉，选择饮食 / 健康 / 菜系标签；已选条件以标签形式展示，可单独删除。
3. **搜索** — 点击**搜索食谱**。切换到其他页面再返回时，搜索结果依然保留。
4. **营养分析** — 点击食谱卡片上的**营养分析**按钮，查看完整营养素分解。
5. **保存食谱** — 点击**保存食谱**将其加入库（供膳食计划使用）。
6. **调整份量** — 输入份数后点击**更新份量**，食材用量自动换算。
7. **清除结果** — 点击页面右上角的**清除结果**按钮可手动重置所有内容。

### 膳食计划

1. **切换周次** — 使用 `←` / `→` 箭头切换，或点击**本周**跳回当前周。
2. **添加餐食** — 在已保存食谱面板中点击**加入日历**，选择日期和餐次类型。
3. **当日热量合计** — 某天有餐食时，日期卡片顶部会显示 🔥 热量标签（所有餐食卡路里之和）。
4. **更新状态** — 通过每条餐食记录上的下拉菜单，将状态标记为已计划 / 进行中 / 已完成。
5. **移除餐食** — 点击餐食记录上的垃圾桶图标。
6. **删除已保存食谱** — 点击已保存食谱面板中食谱卡片上的 🗑️ 图标。

### AI 膳食规划

1. 在膳食计划页面展开 **🤖 AI 膳食规划** 面板。
2. 通过下拉菜单选择**性别**、**年龄**、**身高**和**体重**。
3. 在文本框中描述您的**健康目标**（如"减脂"、"高蛋白"、"低碳水"）。
4. 点击**生成膳食计划** —— AI 将仅使用您已保存的食谱，生成从今天到本周日的计划。
5. 查看预览。点击**重新生成**可获得不同方案；对结果满意后点击**确认并保存到日历**一键写入所有条目。

> ⚠️ 启用 AI 规划功能需要至少 **3 个已保存食谱**。

## API 使用

### Edamam 食谱搜索 API

按食材、饮食标签、健康标签、菜系类型搜索食谱，返回食谱名称、食材列表、制作步骤、份量及每份营养数据。

→ [官方文档](https://developer.edamam.com/edamam-recipe-api)

### Edamam 营养分析 API

对任意食谱计算详细的每种营养素分解（卡路里、宏量营养素、维生素、矿物质），在营养分析弹窗中展示。

→ [官方文档](https://developer.edamam.com/edamam-nutrition-api)

### DeepSeek Chat API

用于 AI 膳食规划功能。接收用户的身体信息、健康目标和已保存食谱列表，返回涵盖本周剩余天数的结构化 JSON 膳食计划。

→ [官方文档](https://platform.deepseek.com/api-docs)
